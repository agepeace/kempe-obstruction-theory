# 具体例子集

## Kempe 链的具体例子

### 例子 1：简单的 (1,3)-Kempe 链

**场景**：考虑一个简单平面图，顶点 v₁, v₂, ..., v₆ 排成一个环，着色为：
```
v₁(颜色1) - v₂(颜色2) - v₃(颜色3) - v₄(颜色1) - v₅(颜色4) - v₆(颜色2) - v₁
```

中间还有边 v₁-v₄，形成"之字形"路径。

**Kempe 链分析**：
- 颜色为 1 或 3 的顶点：v₁(1), v₃(3), v₄(1) 
- 这些顶点形成一个子图。检查连通性：
  - v₁ 和 v₃ 相邻吗？否（中间隔着 v₂）
  - v₁ 和 v₄ 直接相连（存在边）
  - v₃ 和 v₄ 相邻吗？否（中间隔着 v₅, v₆, v₁）
- 因此存在一条 (1,3)-Kempe 链：v₁ - [通过边 v₁-v₄] - v₄，这条链包含 v₁ 和 v₄
- 若要完整描述链，需要找出所有颜色为 1 或 3 的顶点及其 1 或 3 间的路径

**应用**：若我们对这条 (1,3)-Kempe 链进行颜色交换（1↔3），则：
```
v₁(3) - v₂(2) - v₃(1) - v₄(3) - v₅(4) - v₆(2) - v₁
```
仍然是有效的图着色。

## GLFHO 的具体构造

### 例子 2：最小的 GLFHO 构造（9节点极简例）

**图的构造**：
- 中心顶点 w（颜色5）
- w 的五个邻居 u₁, u₂, u₃, u₄, u₅，着色为 1, 2, 3, 4, 2（循环排列）
- 额外的顶点使得形成拓扑反转的结构

**关键结构**：
1. u₁(1) 和 u₃(3) 在同一条 (1,3)-Kempe 链上
2. u₂(2) 和 u₄(4) 在不同的 (2,4)-Kempe 链上
3. Jordan 曲线 Γ = P₁₃ ∪ Q（其中 P₁₃ 是 u₁ 到 u₃ 的 (1,3) 路径，Q 是 u₁→w→u₃ 的通路）分隔平面

**为什么是 GLFHO**：
- 尝试对 u₁ 或 u₃ 所在的 (1,3)-Kempe 链交换：这会改变 w 邻域的颜色结构
- 但无论如何交换，都无法为 w 腾出一种新颜色
- 原因：u₂(2) 和 u₄(4) 被 Γ 分隔，属于不同的 (2,4)-Kempe 链，无法同时"压缩"

### 例子 3：希伍德25节点反例中的 GLFHO

希伍德在1890年构造的反例包含以下关键要素：

**结构特点**：
- 25个顶点的平面图
- 某个5-度顶点 v，颜色为 5，邻居着色为 1,2,3,4,2
- 形成了Kempe链的复杂缠绕

**GLFHO 位置**：
- v 的邻域构成一个 GLFHO
- 两条关键的 Kempe 链在 v 周围形成"缠绕"结构
- 任何简单的 Kempe 交换都无法有效地为 v 补色

## 与五边形的关系

### 例子 4：五边形邻域中的 GLFHO

**构造**：
```
    u₁(1)
   /     \
u₅(2)    u₂(2)
  |   w(5)|
u₄(4)    u₃(3)
   \     /
```

- w 是五边形 u₁u₂u₃u₄u₅ 的中心顶点
- 五边形的顶点是 w 的邻居
- 这正是定理三转移操作的目标

**为什么五边形特殊**：
- 五边形邻域能够形成足够复杂的拓扑结构
- 但又不至于太复杂而无法处理
- 定理一证明了无三/四边形图中至少有两个五边形
- 这两个五边形成为定理三操作中的"源"和"目标"

## 着色与不着色的边界

### 例子 5：4-着色 vs 5-着色

**可以4-着色的情况**：
```
w(1) - u₁(2) - u₂(3) - u₃(4) - u₄(1) - u₅(2) - w(1)
```
虽然 u₅ 的颜色与 w 相同，但在规范的图中，w 和 u₅ 之间没有边，这是合法的。
通过适当的重新着色，这个图总能用4种颜色着色。

**需要5-着色的情况**：
当上述结构中加入额外的边（如 u₁-u₃, u₂-u₄ 等），使得任意4个非 w 顶点两两相邻时，就形成了"Kempe 缠绕"的情况，可能需要第5种颜色。

## 对称拼接的直观理解

### 例子 6：拼接操作的简化模型

**GLFHO 可以转移**：

![GLFHO可以转移](../paper/figures/Transfer1.png)

上图示意了 GLFHO 的可转移性：通过对称拼接和镜像着色，GLFHO 可以从一个五边形邻域转移到另一个五边形邻域。

**假设 U 点为 GLFHO，手动增扩一个五边形面 V**：

![假设U点为GLFHO，手动增扩一个五边形面V](../paper/figures/Transfer2.png)

上图示意了转移操作的第一步：在图中人为插入一个新的五边形面 V，作为 GLFHO 转移的目标位置。

**对图 G 进行对称拼接后，依据定理二，GLFHO 只能消失、或位于 v/v'，原 U 点不再是 GLFHO**：

![对称拼接后GLFHO只能消失或位于v/v'](../paper/figures/Transfer3.png)

上图示意了对称拼接的效果：将 G 与其副本 G' 通过五边形 v 的边界粘合后，依据定理二（GLFHO 唯一性），GLFHO 只能消失、或位于 v/v'，原 U 点不再是 GLFHO。

**原始图 G**：
```
包含GLFHO的图 G
   中心 u 处有 GLFHO
```

**复制与拼接**：
```
   G           G'
(5-着色)  （4-着色）
  +          +
  |←  v作为桥接  →|
  
     拼接后 H
  （对称结构）
```

**着色分配**：
- G 侧：使用 {1,2,3,4,5}，u 处有 GLFHO
- G' 侧：仅使用 {1,2,3,4}，无处有 GLFHO（因为只用4色）
- 拼接后：H 中 G' 侧仍保证无 GLFHO

**移除后**：
- 删除 v
- 只保留 G 侧（原图）
- 由对称性和 G' 侧的4-着色构造，原 G 在新着色下无 GLFHO
- 因此 G 可用4色着色
